# 更新日志

## 2.5.4

- 修复窄侧栏下 Windows 顶部“Help”菜单被主区域分隔线切到的问题。
- 菜单会按内容空间自适应收紧；侧栏达到极限宽度时整组迁移到主区域，避免文字被压缩或遮挡。
- schema v1/v2 运行时新增菜单接缝安全间距自检。

## 2.5.3

- 优化侧栏运行中任务的状态环：缩小视觉尺寸、降低底环对比度，并自动继承当前皮肤强调色。
- schema v1 与 schema v2 皮肤保持一致，同时尊重系统“减少动画”设置。
- 安装器遇到安全软件短暂扫描新文件时会有限退避重试，避免临时目录切换偶发失败。

## 2.5.2

- 修复安装后的 26 套 schema v2 皮肤缺少 WinForms 可读取预览图的问题。
- 安装阶段统一将 WebP 预览转换为缓存 PNG，管理器启动时也会自动修复历史安装中的缺失预览。
- 避免将不受 GDI+ 支持的 WebP 直接交给皮肤列表反复加载。

## 2.5.1

- 修复 125% 及更高 DPI 下安装器、卸载器按钮右侧和底部出现黑色绘制残影的问题。
- 新增 125% DPI 按钮像素级构建回归检查。

## 2.5.0

- 将“动态特效”总开关移至管理器顶部，设置会持久化，并在可热更新时立即同步到当前 Codex 会话。
- 为全部 schema v2 内置皮肤加入通用星光层；关闭特效时移除星光节点并停止主题舞台动画，同时尊重系统“减少动画”设置。
- 重新设计安装器与管理器确认弹窗，安装器默认安装父目录改为 `E:\`。
- 安装过程移至后台线程，避免窗口卡死；缩略图改为小尺寸预缩放缓存，并减少重复皮肤扫描与状态重绘。
- 安装完成后可直接打开管理器，卸载器改为清晰展示删除与保留范围的定制界面。
- 窄窗口自动收紧皮肤库侧栏，后台窗口会暂停星光及舞台动画以降低资源占用。

## 2.4.0

- 离线内置 awesome-codex-skins `skins-v1.1.0` 的全部 26 套认证皮肤，总内置数量增至 29 套。
- 管理器、安装器与 PowerShell 安装流程改为按目录清单播种 schema v1/v2 皮肤。
- 新增上游 Release 同步脚本，校验压缩包路径、大小、格式并记录 SHA-256。
- 安装包自检现在逐套验证全部内置皮肤，并保留上游授权与非商业同人声明。
- 安装器会在解压前检查完整路径预算，并对过深的安装位置给出明确提示。

## 2.3.2

- 修复 Codex 26.715+ 顶部菜单与侧栏、主区域背景断层。
- 将侧栏和主区域处理为连续外壳，只保留最外侧圆角。
- 同步修复 schema v1 与 awesome schema v2 运行时。
- 安装前检测 Codex、管理器和皮肤注入引擎占用。
- 安装成功后旧备份清理失败不再回滚新版本。
- 保留三款内置皮肤并排除 coral-haze。
